# Pokemon GO 自动交易（按键精灵手机助手）

这个仓库包含一个按键精灵手机助手脚本：`auto_trade.mq`。

## 重要修正：不要相信截图红框/固定坐标

前一个版本最大的问题是把截图里的相对坐标当成可靠事实：`CONFIRM` 没点到时可能点到左下角我方宝可梦，导致进入宝可梦详情页。现在脚本按按键精灵文档里的颜色能力重写关键按钮点击：

- `NEXT`：在屏幕下半部中间区域扫描 Pokémon GO 绿色按钮色块，找到后点色块位置。
- `CONFIRM`：只在交易页左侧中下区域扫描绿色按钮色块，避免盲点左下宝可梦。
- 交易完成后的 `X`：只在底部中间区域扫描浅青色圆形关闭按钮。

也就是说：红框、截图里的绝对坐标和 UI 树里那些 LiveCell 坐标不再作为关键按钮依据。脚本仍保留坐标点击交易入口和列表第一格，因为这两步本身就是你指定的固定目标。

## 默认流程

脚本默认从好友详情页开始，也就是能看到 `LOCAL TRADE` / `REMOTE TRADE` / `VIEW TRADE POKÉMON` 的页面：

1. 点击 `LOCAL TRADE`。
2. 等待宝可梦列表加载。
3. 点击左上第一只宝可梦。
4. 不做固定硬等，马上开始扫描绿色 `NEXT` 按钮；`NEXT` 一出现就点击。
5. 点击后验证是否进入交易确认页；如果仍停在 `NEXT` 页，会继续扫描并重试。
6. 扫描并点击左侧绿色 `CONFIRM` 按钮；找不到就停止，避免点到我方宝可梦。
7. 等待对方确认/交易开始。
8. 扫描底部浅青色 `X`，确认交易完成详情页出现后再关闭。
9. 等待回到好友/交易入口页，然后循环下一次。

## 等待时间配置

因为双方经常不同步，默认等待已经整体拉长。打开 `auto_trade.mq` 可以继续调这些变量：

- `WaitPokemonListSec`：点击交易入口后等列表出现。
- `WaitPokemonDetailSec`：选择宝可梦后等待 `NEXT` 按钮出现的最长秒数；脚本不会硬等完整秒数，按钮一出现就会点。
- `WaitTradePageSec`：点击 NEXT 后等确认页出现。
- `WaitPartnerMaxSec`：点 CONFIRM 后等对方确认/交易开始。
- `WaitCompleteMaxSec`：等待交易动画结束、底部 X 出现。
- `WaitReturnSec`：点 X 后等回到好友/交易页。
- `NextRetryMaxSec` / `NextRetryIntervalSec`：`NEXT` 已出现但点击后没有进入交易页时的重试总时长和间隔。

## 后续更稳的方向

颜色扫描比固定坐标稳，但仍然不是最终形态。最稳的方案是：

- 对 `NEXT`、`CONFIRM`、完成页 `X` 做小图附件，用 `FindPic` / `Image.FindPic` 或多点找色确认整个按钮，而不是只看单个绿色像素。
- 如果 Pokémon GO 的 Unity 界面暴露不了可读文本元素，就不要依赖元素树；元素树里重复的 `LiveCell` 对定位按钮帮助有限。
- 每一步都应该是“等待状态出现 → 点击状态内的目标 → 验证进入下一状态”，而不是“睡几秒 → 点某个截图坐标”。
