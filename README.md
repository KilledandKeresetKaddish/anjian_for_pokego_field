# Pokemon GO 自动交易（按键精灵手机助手）

这个仓库现在包含一个按键精灵手机助手脚本：`auto_trade.mq`。

## 从第一性原则拆解流程

自动交易不是“固定点几下”这么简单，核心风险有两个：

1. **状态会等待对方**：点 `CONFIRM` 后，如果对方还没确认，按钮会变成 `CANCEL`。这时继续点击同一区域会取消交易，所以脚本在确认后只等待，不重复点确认。
2. **坐标依赖设备和缩放**：你提供的截图右侧 UI 树显示窗口约为 `960x2079`。脚本把这些坐标作为基准，并通过 `GetScreenX()` / `GetScreenY()` 按比例缩放到当前设备。

## 默认流程

脚本默认从好友详情页开始，也就是能看到 `LOCAL TRADE` / `REMOTE TRADE` / `VIEW TRADE POKÉMON` 的页面：

1. 点击 `LOCAL TRADE`。
2. 在宝可梦列表页点击左上第一只宝可梦。
3. 在详情页按间隔重复点击 `NEXT`，直到超过配置的重试时间；这样第一次没点中时不会直接跳到后面的长等待。
4. 在交易页点击 `CONFIRM`。
5. 等待对方确认。
6. 继续等待交易动画结束，并确保交易完成后的宝可梦详情页有时间出现。
7. 在交易完成后的宝可梦详情页点击底部 `X`。
8. 循环下一次。

## 运行前必须检查的配置

打开 `auto_trade.mq`，优先检查这些变量：

- `MaxTrades`：最大交易次数。第一次建议设为 `1` 或 `3`，确认流程稳定后再提高。
- `WaitPartnerMaxSec`：点 `CONFIRM` 后等待对方确认/交易开始的秒数。对方操作慢时要调大。
- `WaitCompleteMaxSec`：对方确认后，额外等待交易动画结束并进入宝可梦详情页的秒数。动画、网络或加载慢时要调大；脚本会在这个等待结束后才点击底部 `X`。
- `NextRetryMaxSec` / `NextRetryIntervalSec`：控制在宝可梦详情页重复尝试点击 `NEXT` 的总时长和间隔。如果卡在 `NEXT` 页面，优先把 `NextRetryMaxSec` 调大；不要让脚本只点一次就进入后续确认等待。
- `TapTradeEntry()`：默认点击 `LOCAL TRADE`；如果你要点 `REMOTE TRADE`，按脚本注释改坐标。

## 还需要你后续补强的点

当前版本是**坐标状态机**，适合快速试跑。更稳的版本应该继续补图/补色：

- `trade` / `LOCAL TRADE` 按钮图。
- 宝可梦列表第一格或列表页特征图。
- `NEXT` 按钮图，用来确认页面已进入下一步，而不是靠固定时间猜测。
- `CONFIRM` 按钮图。
- 交易完成后底部 `X` 的图。

有了这些附件图后，可以把固定等待改为“找图等待”，效率和稳定性会明显更好。
